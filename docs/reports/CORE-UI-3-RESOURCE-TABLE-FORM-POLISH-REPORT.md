## General UI/UX Improvements
- Navigation grouping dirapikan ke `Master Data`, `Access Control`, `Organization`, `API & System`, dan `Audit / Logs`.
- Table columns diprioritaskan agar nama/kode/status muncul lebih awal.
- Badge ditambahkan/diperjelas untuk identity type, role, app code, role slug, staff type, API ability, HTTP method, dan activity action.
- Filters ditambahkan untuk status, active, role, department, study program, user link, app code, dan API/log fields.
- Searchable/sortable columns ditambah pada nama, kode, email, username, identifier, tanggal, dan status penting.
- Form sections dibuat lebih jelas untuk identity, akademik, penempatan, kontak, akun terhubung, role/akses, permission/periode, dan keamanan.
- Helper text ditambahkan pada field penting seperti username, password, app code, public visibility, role aplikasi, leadership person/unit, dan birth_date.

## Resource Improvements
### Users
- Navigation pindah ke Master Data.
- Table menampilkan nama, username, email, identity badge, roles badge, must-change-password, active, app access count, dan created date.
- Filter role global, identity type, must-change-password, dan active.
- Form diberi helper text untuk nama, username, password, dan role global.

### Students
- Form dibagi menjadi Identitas Mahasiswa, Akademik, dan Akun Terhubung.
- Table menampilkan NIM, nama, email, program studi, linked user, status badge, active, dan created date.
- `birth_date` tetap toggle-hidden by default.
- Filter program studi, status, active, dan akun terhubung.

### Lecturers
- Form dibagi menjadi Identitas Dosen, Penempatan, dan Kontak & Akun.
- Table menampilkan NIDN/NIP, nama, email, department, program studi, linked user, active, dan created date.
- `birth_date` tetap toggle-hidden by default.
- Filter department, program studi, active, dan akun terhubung.

### Employees
- Helper text untuk employee number dan linked user.
- Staff type dan status diberi badge color.
- Existing filters staff type, status, department, dan study program dipertahankan.

### User App Accesses
- Form dibagi menjadi Akses Aplikasi dan Permission & Periode.
- Helper text menegaskan app harus aktif dan role berasal dari app role catalog.
- Table menampilkan user name/email, application, app_code badge, role_slug badge, role name, active, activated/deactivated timestamps.

### Core Applications

### Core Application Roles
