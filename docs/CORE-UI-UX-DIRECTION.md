# Core UI/UX Direction

## Visual Base
- White clean interface as the dominant surface.
- Blue pharmacy/faculty accent for primary actions, section highlights, and page identity.
- Soft sky/cyan/blue-neutral secondary tones for supportive states and quiet emphasis.
- Success, warning, and danger colors remain standard and clear.

## Principles
- Modern, professional, and easy to scan.
- Avoid pale/plain pages by using clear hierarchy, section rhythm, and blue accent lines.
- Avoid overwhelming color; white cards and readable text remain the base.
- Use consistent cards, badges, and empty states.
- Keep tables readable with enough spacing and sticky mental structure.
- Keep responsive layouts usable on tablet/mobile.

## Component Direction
- Summary cards should show metric label, value, and status clearly.
- Problem or warning counts should be visible without relying only on color.
- Empty states should explain the current state and next safe action.
- Sensitive actions such as execute, rollback, rotate, revoke, and delete require confirmation and visual warning.
- Import pages should clearly show lifecycle steps: template, upload, preview, decision, execute, rollback.

## Security UX
- Do not present Core as public.
- Do not imply SSO or auto-login.
- Do not expose password, token, secret, or hash.
- Keep destructive and high-risk operations visually distinct.
- Keep read-only dashboards clearly labeled as read-only.
