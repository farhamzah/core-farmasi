# CORE-UI-4 Light Theme & Browser Visual QA Report

## Scope
Tahap ini mengubah admin Core Farmasi UBP agar default ke light theme putih bersih dengan aksen biru farmasi/fakultas, lalu melakukan browser visual QA dan responsive check ringan. Perubahan ini bukan database change dan bukan perubahan logic bisnis, auth, API, import, rollback, atau password flow.

## Previous Reports Reviewed
- CORE-UI-2 Admin Dashboard & Blue Pharmacy UI Polish.
- CORE-UI-3 Resource Table/Form Polish.
- CORE-UI-UX-DIRECTION.

## Initial Visual Problem
- Core masih bisa tampil dominan dark/hitam saat browser atau OS memakai dark color preference.
- Custom page menggunakan beberapa class `dark:*` sehingga Data Quality Dashboard, Import Center, dan App Launcher ikut menjadi gelap pada environment tertentu.
- Setelah `viteTheme()` dipasang, `resources/css/app.css` sempat belum mengimpor stylesheet utama Filament sehingga layout admin tampil hampir mentah dan ikon besar muncul di halaman admin.
- Target owner adalah white/light base dengan aksen biru farmasi, bukan dark/black default.

## Theme Changes
- Filament admin panel dikonfigurasi memakai `resources/css/app.css` sebagai Vite theme.
- `resources/css/app.css` kini mengimpor `vendor/filament/filament/resources/css/theme.css` terlebih dahulu agar base/component stylesheet Filament tetap lengkap.
- Dark mode panel Core dinonaktifkan melalui `darkMode(false)` agar default admin tetap light.
- CSS theme menambahkan:
  - white/light blue admin background,
  - blue pharmacy/faculty accent variables,
  - white sidebar and topbar treatment,
  - blue active/hover navigation state,
  - white card/table/form surfaces,
  - soft blue borders and shadows,
  - primary button blue gradient,
  - table containment for responsive width.
- Karena Tailwind `dark:*` masih bisa aktif dari browser dark preference, CSS juga menambahkan override khusus dalam `prefers-color-scheme: dark` agar Core tetap light.

## Issues Found
- Dark preference browser membuat custom pages tertentu tetap memakai dark background dari class `dark:*`.
- Beberapa resource table bisa memicu horizontal page overflow di tablet/mobile sebelum containment CSS ditambahkan.
- Filament base stylesheet sempat tidak ikut masuk ke custom Vite theme; ini menyebabkan tampilan admin mentah dan ikon/shape besar.
- Tidak ditemukan asset 404.
- Tidak ditemukan 500 error.
- Tidak ditemukan console error pada browser sweep.

## Fixes Applied
- Light theme default diterapkan di Filament panel.
- Filament base theme diimpor kembali ke `resources/css/app.css`.
- Dark mode admin Core dinonaktifkan untuk konsistensi visual.
- Sidebar/header/topbar dipoles ke white surface dengan blue active state.
- Card, table, modal, dropdown, form, input, select, textarea dipoles ke white surface dan readable dark text.
- Custom page dark variants dioverride agar tetap light pada OS/browser dark preference.
- Table/resource containers diberi containment agar mobile/tablet tidak menghasilkan horizontal overflow besar.
